# CJ 팔레타이징 — PCT 개선 계획 (파라미터 & 코드)

> 현재 상태: PCT(setting3, 실제밀도) 학습 중. 제출 best 점수 **87.80** (baseline 16.35).
> 약점: 적재율이 낮아서가 아니라 **가끔 가장자리 박스 1개가 경계 이탈 → 그 파일 0점**.
> 따라서 "적재율 올리기"보다 **"0점 안 나게(견고성)"** 가 우선.
> 관련 문서: `Online-3D-BPP-PCT/CJ_README.md`(작업 가이드)

---

## 우선순위 요약

```
1순위: ① 안전마진(xy_margin)   ← 즉시, 재학습X, 0점 방지            ★최시급
2순위: ⑤ 학습 더 + best 사용    ← cj3 돌리는 중, PCT-best.pt 활용
3순위: ② leaf_node_holder ↑     ← 후보 늘려 적재율 약간↑
4순위: ⑧ 보상에 물리 반영        ← 근본 해결(재학습+env수정, 큰 작업)
실험:  ④ 네트워크 키우기
상시:  ⑩ 여러 체크포인트 Isaac 비교로 최종 선택
```

핵심 한 줄: **①안전마진이 1순위.** 점수를 깎는 건 적재율이 아니라 "가끔 1박스 이탈로 0점"이라서.

## 🟢 1. 추론·견고성 (재학습 불필요, 가성비 최고)

### ① 안전 마진 `xy_margin` ★최우선
- **무엇**: 팔레트보다 살짝 작은 영역에 패킹 → 박스를 경계에서 떨어뜨림. 물리 정착 중 밀려도 경계 안 넘게.
- **왜**: box_2 0점의 직접 원인(경계 끝까지 채워 박스가 미끄러져 이탈). 허용오차 0.02m.
- **어디**: `templete code/config/pct_config.yaml`에 `xy_margin: 0.02` 추가 + `templete code/algorithm.py`에서 컨테이너를 `[L-2m, W-2m, H]`로 쓰고 출력 좌표 x,y에 +m offset.
- **난이도**: 낮음 (재학습 X). **효과**: 0점 위험 대폭↓ (적재율 몇 % 손해 << 0점 방지).
- **상태**: 아직 미적용 (제안만 함).

### ② leaf_node_holder (후보 수) 늘리기
- **무엇**: 박스마다 고려하는 배치 후보 수(현재 100) 증가 → 더 좋은 위치 선택 여지.
- **어디**: `pct_config.yaml`의 `leaf_node_holder`. ⚠️ **학습/ONNX export/추론 세 곳 값이 동일해야 함** → 늘리면 재학습 필요.
- **난이도**: 중 (값 통일). **효과**: 적재율 소폭↑, 속도 약간↓(90초 여유 있음).

### ③ 가장자리 후보 down-rank
- **무엇**: 정책이 고른 위치가 "경계 + 지지 부족"이면 차순위 안전 후보로 교체.
- **어디**: `templete code/algorithm.py`의 잎 선택 로직(argmax 뒤 후처리).
- **난이도**: 중. **효과**: 견고성↑ (마진과 병행하면 더 안전).

## 🟡 2. 학습 하이퍼파라미터 (재학습 필요)

### ④ 네트워크 키우기 (실험적)
- **무엇**: 현재 모델 236KB로 매우 작음(embed 64, hidden 128, **GAT 1층**). `--gat-layer-num 2~3`, `--embedding-size 128`로 키우면 복잡한 패킹 학습 가능.
- **어디**: train 명령 인자 (`train_cj3.sh` 수정).
- **난이도**: 중(모델 커지고 학습 더 오래). **효과**: 불확실(향상 가능, 실험 필요).

### ⑤ 더 길게 학습 + best 체크포인트
- **무엇**: 충분히 돌리고 `PCT-best.pt`(이미 best-save 적용됨) 사용.
- **어디**: `train_cj3.sh` 계속 / `--load-model`로 이어서. monitor.sh로 수렴 확인.
- **난이도**: 낮음(시간). **효과**: 적재율↑ (점수 직결).

### ⑥ num-processes
- **무엇**: 16 → 32로 늘리면 그래디언트 안정(CPU 여유 보고).
- **어디**: train 명령 `--num-processes`. **효과**: 학습 안정성 소폭↑.

### ⑦ learning-rate (신중)
- **무엇**: 1e-6 (ACKTR이라 민감). 섣불리 건들면 불안정.
- **어디**: `--learning-rate`. ⚠️ 위험, 마지막에.

## 🔴 3. 코드/보상 개선 (큰 작업, 근본 해결)

### ⑧ 보상에 물리 반영 ★근본책
- **무엇**: 현재 보상 = 부피(적재율)뿐. **지지 부족/가장자리 배치에 패널티** 추가 → 모델이 "안 무너지는 배치"를 직접 학습.
- **어디**: `Online-3D-BPP-PCT/pct_envs/PctContinuous0/bin3D.py`의 `reward = box_ratio*10` + `space.py` 안정성.
- **난이도**: 높음(보상 설계 + 재학습). **효과**: 0점 근본 차단, 견고성 큰 향상.

### ⑨ support 임계값 강화
- **무엇**: PCT 안정성 검사를 더 빡세게(가장자리 박스에 더 많은 지지 요구).
- **어디**: `pct_envs/PctContinuous0/space.py`의 stability 체크.
- **난이도**: 중~높음. **효과**: 견고성↑, 적재율은 약간↓ 가능.

## 🔵 4. 프로세스 개선 (상시)

### ⑩ 여러 체크포인트 Isaac 비교
- **무엇**: 기하 적재율 ≠ 물리 점수, latest ≠ best. 후보 여러 개를 Isaac으로 돌려 **실제 점수 최고**를 최종 선택.
- **어디**: `export_to_contest.py`(빠른 테스트) → `palletizing_simulator/simulator.py`.
- **난이도**: 낮음(반복). **효과**: 안전한 최종 선택.

## 권장 진행 순서

1. **지금**: cj3 학습 계속 (⑤) + `bash monitor.sh`로 수렴 확인.
2. **언제든**: ① 안전마진 적용 (재학습 무관, 제출 안정성 즉시↑).
3. **cj3 수렴 후**: best 체크포인트 → ONNX(setting3) → 제출물 통합 → ⑩ Isaac 비교 (setting1 87.80 vs setting3).
4. **여유되면**: ② leaf_holder↑ 또는 ④ 네트워크↑ 로 재학습 실험.
5. **시간 많으면**: ⑧ 보상에 물리 반영 (근본 견고성).

## 참고: 절대 잊지 말 것
- **"1박스 실패 = 그 파일 0점"** → 견고성(①⑧⑨)이 적재율보다 중요.
- holder(internal/leaf) 값은 **학습=export=추론 3곳 일치** 필수.
- 회전은 setting1/3 모두 0/90만 (대회와 일치).
- 제출물은 torch 금지 → **ONNX만**. 처리시간 90초 (현재 ~5초, 여유 큼).
